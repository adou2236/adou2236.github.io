背景

开发中遇到一个这样的问题：一个提交的表单数据新增功能现在添加将默认数据回填再进行修改的功能，代码大概写成这样

然后问题就来了，测试的过程中发现某个字段（remark）无法进行编辑，但是该字段在空白表单下却是可以编辑的。只是在通过上述方式赋值之后便无法编辑了，遂排查。

解决

通过监听表单项的input事件发现是this.formData.remark是变化的，只是页面视图没有更新，所以问题就明确了，this.formData.remark失去了响应性

发现在 this.formData初始化时并没有给remark赋初始值，添加上初始值后便能正常工作。但是为什么在空白表单编辑时却从来没发生过问题，该字段的数据也能正常提交。

查询资料后发现，在给v-model绑定某个对象上不存在的属性时，会静默地将这个属性添加到该对象上，并为其创建getter与setter使其具备响应性。
这就是为什么以往的新增功能可以正常工作

引申

v-model是如何双向绑定一个对象中不存在的属性的。

1、属性添加：
当你使用 v-model 绑定一个对象中不存在的属性时，Vue.js 会在组件实例中动态地添加这个属性。这个操作并不会触发 Vue 的响应式系统。

2、监听输入事件：
Vue.js 会为绑定了 v-model 的表单元素添加一个 input 事件监听器。当用户在表单元素中输入内容时，input 事件会被触发。

3、更新属性值：
当 input 事件被触发时，Vue.js 会将表单元素的新值更新到 Vue 实例的数据对象中，包括动态添加的属性。虽然这个属性并不在组件实例的初始数据中，但 Vue.js 仍然能够将它添加到响应式系统中，并确保在属性值发生变化时更新视图。
